# UMI-Manipulation-in-Construction — Software

End-to-end software stack for UMI-based construction robotics:
data collection → calibration → trajectory replay → diffusion-policy training → rollout.

This folder contains all source code, configs, calibration data, and the MuJoCo
xArm7 model. Large artifacts (raw demonstrations, trained checkpoints, RGB/Depth
frames, videos) are hosted on Google Drive — see [External assets](#external-assets) below.

---

## Folder structure

| Path | Purpose |
|------|---------|
| `train/` | Diffusion-policy training (`train_umi_ft.py`), rollout (`run_umi_ft.py`), dataset class, vision encoder, evaluation scripts. Includes a vendored `diffusion_policy/` library. |
| `models/ufactory_xarm7/` | MuJoCo XML model + meshes for the xArm7 robot. Used by `replay_trajectory.py --sim` and `sim_replay.py`. |
| `calibration/` | Hand-eye and ArUco calibration JSON. Generated by `calibrate_hand_eye_*.py`. |
| `config/` | Calibration / runtime configs (e.g., `aruco_config.yaml`). |
| `umi_ft/` | UMI-FT helper utilities (ArUco detector, gripper calibration). |
| `Demonstration.py` | iPhone Record3D RGBD capture script (data collection). |
| `replay_trajectory.py` | Replay a CSV trajectory on xArm7 (with optional MuJoCo preview). |
| `replay_ee_pose.py` | Replay normalized EE-pose trajectories. |
| `replay_with_recording.py` | Replay + record RGBD for verification. |
| `generate_ee_pose_csv.py` | Convert raw camera trajectories to body-frame EE poses. |
| `crop_episodes.py`, `trim_trajectory.py`, `downsample_episodes.py` | Data preprocessing utilities. |
| `calibrate_*.py` | Calibration scripts (gripper width, hand-eye, UMI↔xArm). |
| `sim_replay.py` | MuJoCo-only replay for verification. |

## Quickstart

### Install
```bash
conda create -n umi_ft python=3.10 -y
conda activate umi_ft
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install diffusers timm einops opencv-python pillow scipy mujoco \
            record3d xarm-python-sdk tqdm wandb
```

### Download data + checkpoints from Google Drive
See [External assets](#external-assets). Place the unzipped data under
`Data_Raw/` and `best.pt` under `train/checkpoints/`.

### Generate EE-pose CSVs
```bash
python generate_ee_pose_csv.py
```
This creates `ee_pose_trajectory.csv` next to every `trajectory.csv`
in `Data_Raw/data_crop/`.

### Train
```bash
cd train
python train_umi_ft.py
```
Adjust `cfg["data_dir"]`, `batch_size`, `num_epochs` in `train_umi_ft.py`
to match your hardware.

### Rollout on xArm7
```bash
cd train
python run_umi_ft.py
```
By default loads `D:\UMI_Gripper\train\checkpoints\best.pt`.
Update `CKPT_PATH` in `run_umi_ft.py` to match your local path.

## External assets

Because Github limits files to 100 MB and repositories to ~5 GB, large
artifacts are stored on Google Drive:

**Drive folder**:

Contents you need to download:

| File | Local path | Description |
|------|-----------|-------------|
| `Data_Raw.zip` | `Data_Raw/` | All collected RGBD demonstrations (~600 GB uncompressed). Contains `data/` (raw) and `data_crop/` (gripper-open-only crops used for training). |
| `best.pt` | `train/checkpoints/best.pt` | UMI-FT policy checkpoint (epoch 219, val_naction_mse 0.0228). Used by `run_umi_ft.py`. |

| `Model_Result.zip` *(optional)* | `models/Model_Result/` | Older / experimental checkpoints. Not needed for the main pipeline. |

## Data layout (after download)

```
Data_Raw/data_crop/episode_XXXX_TIMESTAMP/
├── rgb/000000.png   ...   000NNN.png   # 1920×1440 RGB frames
├── depth/000000.png ...   000NNN.png   # 256×192 depth frames
├── rgb.mp4                              # RGB video (for inspection)
├── depth.mp4                            # Depth video (for inspection)
├── trajectory.csv                       # iPhone camera pose + image paths
└── ee_pose_trajectory.csv               # Body-frame accumulated EE pose
                                         # (generated by generate_ee_pose_csv.py)
```

`trajectory.csv` columns: `timestamp, tx, ty, tz, qx, qy, qz, gripper_width, rgb_path, depth_path`
(despite the `qx/qy/qz` names, those are rotation-vector components in radians)

`ee_pose_trajectory.csv` columns: same, but `tx/ty/tz` are EE position in meters
and `qx/qy/qz` are EE rotation-vector in radians, accumulated from identity.

## Pipeline at a glance

```
[Demonstration.py]
   iPhone Record3D capture
        ↓ rgb.mp4 / depth frames / trajectory.csv
[crop_episodes.py]
   keep only frames where gripper is open
        ↓ Data_Raw/data_crop/
[generate_ee_pose_csv.py]
   camera pose → hand-eye → body-frame EE
        ↓ ee_pose_trajectory.csv
[train/train_umi_ft.py]
   ViT-B/32 CLIP + modality-attention + DDIM diffusion UNet
        ↓ train/checkpoints/best.pt
[train/run_umi_ft.py]
   Live Record3D → policy.predict_action → xArm7 servo command
```
